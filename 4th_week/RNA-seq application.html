Data

The RNA-seq data that we will use in this class can be found at /lustre1/share/rnaimmune/data

In this folder, there are two subfolders, expression and follow_up. The Matrix in expression is a count matrix, with rows as genes and columns as samples. The Matrix in follow_up records metadata of samples.

The RNA-seq data was generated by (Van Allen et al., 2015). It contains whole-transcriptome sequencing of tumor biopsies of 42 patients with metastatic melanoma before ipilimumab (anti-CTLA-4) treatment.

B. Gene-to-gene correlation

We analyze the correlation between two specific genes: GZMA (Granzyme A) and PRF1 (Perforin 1). These two genes indicate immune cytolytic activity (CYT).

library(Hmisc)
library(ggplot2)

gene1 <- 'GZMA'
gene2 <- 'PRF1'

pearson_corr <- rcorr(exprsn[gene1,], exprsn[gene2,], type = 'pearson')
spearman_corr <- rcorr(exprsn[gene1,], exprsn[gene2,], type = 'spearman')

print(spearman_corr)
  • Pearson correlation measures the linear relationship between two genes.
  • Spearman correlation captures monotonic relationships, accounting for ranks instead of actual values.

We plot the expression levels of the two genes to visualize their correlation.

df <- data.frame(t(rbind(exprsn[gene1,], exprsn[gene2,])))
colnames(df) <- c('gene1', 'gene2')

spearman_label <- paste0('Spearman ρ = ', round(spearman_corr$r[1,2], 2), ', p = ', round(spearman_corr$P[1,2], 4))

# scatter plot
p <- ggplot(df, aes(gene1, gene2))
p <- p + geom_point()
p <- p + annotate("text", x = 50, y = 13, label = pearson_label, size = 5)
p <- p + annotate("text", x = 50, y = 10, label = spearman_label, size = 5)
p <- p + xlab(gene1)
p <- p + ylab(gene2)

C. Cox proportional hazards model

Finally, we use the Cox proportional hazards model to evaluate the effect of multiple variables on survival, including age, gender, cancer stage, CYT level, and CTLA4 expression level. This model provides hazard ratios (HR) for each variable, indicating their influence on overall survival.

The Cox model:

h(t)=h0(t)×exp(b1x1+b2x2+⋯+bpxp)h(t) = h_{0}(t) \times exp(b_{1}x_{1} + b_{2}x_{2} + \cdots + b_{p}x_{p})h(t)=h0​(t)×exp(b1​x1​+b2​x2​+⋯+bp​xp​)

The harzard ratio of xix_{i}xi​ is then exp(bi){exp(b_{i})}exp(bi​)

metadata$CTLA4 <- exprsn['CTLA4',]
metadata$Gender <- factor(metadata$Gender, levels=c(0,1))
levels(metadata$Gender) <- c('Female', 'Male')
cox_model <- coxph(Surv(OS, `OS.Event`) ~ Age + Gender + Stage + CYT + CYT_level + CTLA4, data = metadata)

Note: In R, the ~ symbol is used to specify a formula, which is commonly used in statistical modeling and plotting functions.

Have a look at the regression results with summary.

The forest plot visually summarizes these results.

Part 3. Immune infiltration analysis

A. Deconvolute immune cell fractions 

library(tidyverse)
library(pheatmap)
library(immunedeconv)
set_cibersort_binary("/lustre1/share/rnaimmune/CIBERSORT.R")
set_cibersort_mat("/lustre1/share/rnaimmune/LM22.txt")

To estimate immune cell fractions, we have to invoke the deconvolute function. It requires the specification of one of the following methods for deconvolution:

print(deconvolution_methods)

The deconvolute function produces a data.frame with the first column containing the cell-type names and the other columns containing the estimates for each sample.

To interpret the results, it is very important to understand the different scoring strategies of the methods. EPIC, CIBERSORT and quanTIseq are the only methods providing an "absolute score" that represents a cell fraction. All other methods provide scores in arbitrary units, which are only meaningful in relation to another sample of the same dataset.

# MCP-counter
res_mcp <- deconvolute(exprsn, "mcp_counter")
# EPIC
res_epc <- deconvolute(exprsn, "epic")
# quanTIseq
res_qts <- deconvolute(exprsn, "quantiseq")
# xCell
res_xcl <- deconvolute(exprsn, "xcell")
# CIBERSORT
res_cbs <- deconvolute(exprsn, "cibersort") # will save results in the work dir
# CIBERSORT abs. mode
res_cba <- deconvolute(exprsn, "cibersort_abs")
# TIMER
res_tmr <- deconvolute(exprsn, "timer", indications = rep('skcm', ncol(exprsn)))

Excluding genes with low cell-type specificity can reduce spillover-effects and improve overall results. If you find a gene to impair the results, you can exclude it using the rmgenes parameter. Here we remove six genes that are expressed by both B cells and plasmacytoid dendritic cells (pDCs).

res_qts <- deconvolute(
    exprsn, method='quantiseq', 
    rmgenes = c('TCL1A', 'TCF4', 'CD37', 'SPIB', 'IRF8', 'BCL11A'))

To be able to run TIMER, you have to provide the proper tumor-type to the deconvolute function. The full list of abbreviations is available on the TCGA wiki and stored in timer_available_cancers.

print(timer_available_cancers)

Have a glance at the results. Each contains a tibble with the first column as cell types.

print(res_qts)

It seems that TIDE gives accurate prediction, but how to quantify it?

ROC (Receiver Operating Characteristic) curve is a graphical plot used to evaluate the performance of a binary classification model. It shows the trade-off between the true positive rate (sensitivity) and the false positive rate (1-specificity) at different threshold settings.

AUC (Area Under the Curve) refers to the area under the ROC curve. It provides a single metric to assess the model's performance. AUC values range from 0 to 1, where:

  • 1 indicates a perfect model,
  • 0.5 means the model is no better than random chance,
  • 0 reflects a completely incorrect model.

In general, the higher the AUC, the better the model's ability to distinguish between the two classes.

library(pROC)

TIDE_roc <- roc(Responder ~ TIDE, data=res_tide, aur=TRUE)

# ROC plot
p <- ggroc(TIDE_roc, legacy.axes = TRUE)
p <- p + geom_segment(x = 0, xend = 1, y = 0, yend = 1 ,color="darkgrey", linetype=4)
p <- p + theme_bw()
p <- p + annotate("text",x=0.75,y=0.125,label = paste("AUC =", round(TIDE_roc$auc,3)))
png('roc.png')
print(p)
dev.off()

The ROC curve can be smoothed via a binormal smoothing.

Note: The binormal model assumes that all variables are continuous and that there is a function that associates a score with each individual. The distribution of scores is assumed to be normal for the positive and negative populations.

# smooth the roc curve
TIDE_roc_smooth <- roc(Responder ~ TIDE, data=res_tide, aur=TRUE, smooth=TRUE)

# ROC plot
p <- ggroc(TIDE_roc_smooth, legacy.axes = TRUE)
p <- p + geom_segment(x = 0, xend = 1, y = 0, yend = 1 ,color="darkgrey", linetype=4)
p <- p + theme_bw()
p <- p + annotate("text",x=0.75,y=0.125,label = paste("AUC =", round(TIDE_roc_smooth$auc,3)))
png('roc_smooth.png')
print(p)
dev.off()

Note: the AUC has been changed after smoothing.
